FileNameFilter is a class, designed to filter file names according a set of usual meta-templates with wildcards '*' and '?'. 


class FileNameFilter : NoCopy
 {
   ....

  public:

   FileNameFilter() noexcept;

   explicit FileNameFilter(StrLen filter,ulen max_states=MaxULen);

   ~FileNameFilter();

   // props

   bool operator + () const;

   bool operator ! () const;

   // methods

   void reset();

   void reset(StrLen filter,ulen max_states=MaxULen);

   bool operator () (StrLen file) const;
 };

Default constructor creates an object in null state. You can reset an object to this state using the method reset(). Null state object cannot be used to filter file names. You have to initialize it first.

Non-default constructor creates an initialized object. filter is a file name template with wildcards, max_states is a number of states limit. Constructor creates a state machine to examine file names.

operator + () is true iff the object is initialized.

operator ! () is true iff the object is in null state.

reset() return the object into the null state.

reset() with arguments reinitializes the object.

operator () checks, if the given file name matches the file name templates. If the object is not initialized, false is returned.
